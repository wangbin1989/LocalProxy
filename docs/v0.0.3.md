# v0.0.3 — 代理配置管理命令

## 概述

在 v0.0.2 配置文件支持基础上，新增代理配置管理命令组，用于增删改查 JSON 配置文件中的代理条目。新增 `Enabled` 字段支持启用/停用状态切换。

## 命令规格

```
localproxy add <name> -l <port> -H <host> -p <port> -P <protocol> --config <file>
localproxy update <name> [-l <port>] [-H <host>] [-p <port>] [-P <protocol>] --config <file>
localproxy remove <name> --config <file> [--force]
localproxy enable <name> --config <file>
localproxy disable <name> --config <file>
localproxy list --config <file>
```

### 命令详情

| 命令 | 说明 |
|---|---|
| `add` | 添加代理配置，校验名称唯一性 |
| `update` | 更新已有代理，仅更新指定的字段 |
| `remove` | 删除代理，`--force` 跳过确认 |
| `enable` | 启用已停用的代理 |
| `disable` | 停用代理（保留配置） |
| `list` | 表格展示所有代理及状态 |

## 配置服务

`ConfigService` 提供完整的配置文件读写操作：

| 方法 | 说明 |
|---|---|
| `LoadAsync(path)` | 加载配置列表（文件不存在返回空列表） |
| `SaveAsync(path, configs)` | 保存配置列表（自动创建目录） |
| `AddAsync(path, config)` | 添加条目（校验名称唯一性） |
| `UpdateAsync(path, name, config)` | 按名称更新条目 |
| `RemoveAsync(path, name)` | 按名称删除条目 |
| `SetEnabledAsync(path, name, enabled)` | 设置启用/停用状态 |
