# Sprint 1 — TCP 代理核心

**周期：** 第 2-3 周（2 周）  
**目标：** 实现 TCP 连接转发引擎的核心功能，包含基本日志

---

## 冲刺目标

实现 `LocalProxy.Engine` 项目，提供 TCP 端口转发能力，支持多并发连接、优雅关闭和字节统计日志。

---

## 用户故事

### ENG-001: TCP 端口转发（8 点）

> 作为开发者，我希望将本地端口 X 通过 TCP 转发到远程 host:port Y。

**验收标准：**
- 在本地端口 9000 启动代理，转发到 `example.com:80`
- `curl localhost:9000` 返回 example.com 的内容
- 关闭代理后端口释放

**技术要点：**
- 使用 `TcpClient` / `TcpListener` 实现双向数据复制
- 异步 I/O：`async/await` 全链路
- 实现 `ITunnelHandler` 接口

### ENG-002: 并发连接处理（5 点）

> 作为开发者，我希望代理隧道支持多个并发连接。

**验收标准：**
- 50 个并发连接到转发端口，全部完成，数据无损坏
- 连接数可配置上限，超过上限时拒绝新连接并记录日志
- 连接断开后资源立即释放

### ENG-003: 优雅关闭（3 点）

> 作为开发者，我希望连接关闭时优雅释放资源。

**验收标准：**
- 循环启动/停止 1000 次，无端口耗尽，无 socket 异常
- 停止时等待已有连接完成（可配置超时），再释放端口
- `StopAsync` 调用后不再接受新连接

### ENG-004: 字节统计日志（3 点）

> 作为运维，我希望每个连接记录字节数日志。

**验收标准：**
- 隧道停止时日志记录本次会话的总收发字节数
- 日志级别为 Information
- 日志包含隧道名称、本地端口、远程地址
