## 用户故事

### BLD-006: 安装说明（2 点）

> 作为用户，我希望每个平台有清晰的安装说明。

**验收标准：**
- README.md 更新各平台安装步骤
- macOS：DMG 安装 + `brew` 安装说明
- Windows：MSIX 安装 + 便携版说明
- Linux：`.deb`/`.rpm` 安装 + tar.gz 手动安装说明
- 每种方式包含验证步骤（`localproxy --version`）

### BLD-007: 贡献指南（3 点）

> 作为开发者，我希望有贡献指南文档。

**验收标准：**
- `CONTRIBUTING.md` 包含：
  - 开发环境搭建（dotnet SDK 版本、IDE 推荐）
  - 构建步骤（Debug / Release）
  - 运行测试（单元 + 集成）
  - 代码风格约定
  - PR 提交流程
  - Issue 模板

### BLD-008: FAQ 文档（3 点）

> 作为用户，我希望有常见问题文档。

**验收标准：**
- `docs/faq.md` 覆盖常见问题：
  - 端口冲突怎么办？
  - 如何查看日志？
  - 代理连接失败如何排查？
  - 如何设置开机自启？
  - 配置文件的格式和位置？
  - 如何升级版本？
  - 如何卸载？

## 打磨任务清单

### 全平台验收测试

- [ ] Ubuntu 24.04 (arm64, 树莓派)：CLI 基本功能

### 缺陷修复优先级

1. **P1（阻塞发布）：** 崩溃、数据丢失、安全漏洞 → 必须修复
2. **P2（影响体验）：** UI 异常、日志缺失、错误信息不清晰 → 尽量修复
3. **P3（可延后）：** 偶发问题、边缘场景 → 记录到后续迭代

### 性能验证

- [ ] TCP 转发吞吐量 >= 基线 90%
- [ ] UDP 转发延迟 < 5ms（本地回环）
- [ ] GUI 内存占用 < 200MB（macOS 空闲状态）
- [ ] 服务进程内存占用 < 50MB（10 条隧道运行）

### 用户体验打磨
