# v0.0.1 — run 命令

## 概述

首次迭代，实现 `run` 子命令，用于快速启动单个端口代理。接收本地端口、远程地址、远程端口和协议类型，校验后启动代理转发，前台阻塞运行，Ctrl+C 停止。

## 命令规格

```
localproxy run -l <port> -H <host> -p <port> -P <tcp|udp|http>
```

### 选项

| 选项 | 别名 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| `--local-port` | `-l` | `int` (0–65535) | 是 | 本地监听端口 |
| `--remote-host` | `-H` | `string` | 是 | 远程目标主机地址（IP 或域名） |
| `--remote-port` | `-p` | `int` (0–65535) | 是 | 远程目标端口 |
| `--protocol` | `-P` | `tcp \| udp \| http` | 是 | 代理协议类型 |

### 行为流程

1. 解析命令行参数
2. 校验：端口范围、协议值、主机地址非空（全部校验后再输出错误）
3. 通过 Spectre.Console 面板输出代理配置摘要
4. 启动代理转发，前台阻塞运行
5. 输出连接日志（客户端地址 → 远程地址）
6. Ctrl+C 优雅停止，输出 "代理已停止"
7. 退出，状态码 0（成功）或 1（校验失败）

### 输出示例

```
         代理已启动          
╭──────────┬────────────────╮
│ 本地地址 │ :8080          │
│ 远程地址 │ example.com:80 │
│ 协议类型 │ TCP            │
╰──────────┴────────────────╯
信息: TCP 代理已启动，监听端口 :8080
连接: 新连接 127.0.0.1:54321 → example.com:80

# HTTP 代理输出示例：
信息: HTTP 代理已启动，监听端口 :8080
连接: HTTP GET /api/users → example.com:80
```

### TCP 代理

- 使用 `TcpListener` 监听本地端口
- Accept 循环接收客户端连接
- 每个客户端建立到远程的 `TcpClient`
- 双向 `Stream.CopyToAsync` 桥接数据
- 连接日志格式：`新连接 客户端地址 → 远程地址`

### HTTP 代理

- 基于 TCP，读取 HTTP 请求行解析方法（GET/POST 等）和路径
- 转发请求行并双向中继剩余数据
- 连接日志格式：`HTTP GET /path → 远程地址`

### UDP 代理

- 使用 `UdpClient` 监听本地端口
- Receive 循环接收数据报
- 为每个数据报创建临时 `UdpClient` 转发到远程
- 接收远程响应后回传客户端
