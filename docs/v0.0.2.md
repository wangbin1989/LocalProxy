# v0.0.2 — run --config 配置文件

## 概述

在 v0.0.1 单代理 CLI 模式基础上，新增 `--config` 选项，支持从 JSON 配置文件读取代理配置。配置文件可包含多个代理定义，一次性全部启动。CLI 模式保持向后兼容。

## 命令规格

```
localproxy run [--config <path>] [-l <port>] [-H <host>] [-p <port>] [-P <protocol>]
```

### 选项

| 选项 | 别名 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| `--config` | — | `string` (路径) | 否 | JSON 配置文件路径 |
| `--local-port` | `-l` | `int` (0–65535) | 否 | 本地监听端口 |
| `--remote-host` | `-H` | `string` | 否 | 远程目标主机地址 |
| `--remote-port` | `-p` | `int` (0–65535) | 否 | 远程目标端口 |
| `--protocol` | `-P` | `tcp \| udp \| http` | 否 | 代理协议类型 |

### 两种运行模式

**配置文件模式（`--config`）：**
1. 读取 JSON 配置文件，解析为代理列表
2. 校验所有代理配置
3. 多代理表格展示启动面板
4. 并发启动全部代理
5. Ctrl+C 停止所有代理

**CLI 模式（无 `--config`）：**
行为与 v0.0.1 一致，四个选项必填，启动单个代理。

## 配置文件格式

JSON 数组，每个元素定义一个代理：

```json
[
  {
    "name": "web-proxy",
    "localPort": 8080,
    "remoteHost": "example.com",
    "remotePort": 80,
    "protocol": "tcp"
  },
  {
    "name": "api-proxy",
    "localPort": 9090,
    "remoteHost": "api.example.com",
    "remotePort": 443,
    "protocol": "http"
  }
]
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | `string` | 代理名称 |
| `localPort` | `int` | 本地监听端口 |
| `remoteHost` | `string` | 远程目标主机 |
| `remotePort` | `int` | 远程目标端口 |
| `protocol` | `tcp \| udp \| http` | 代理协议（大小写不敏感） |
