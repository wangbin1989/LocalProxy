# v0.0.6 — UDP 地址解析修复与 CLI 默认协议

## 概述

1. 修复 UDP 代理中 `Dns.GetHostAddresses(...).First()` 的两个问题：空数组异常和 IPv4/IPv6 地址族不匹配
2. 修复 `add` 命令未提供 `--protocol` 时的异常

## 变更内容

### 修改

- `ProxyService.StartUdpProxyAsync` — DNS 解析改为优先匹配地址族，失败时抛出明确的 `InvalidOperationException`
  - 优先选择 IPv4 地址与本地回环地址匹配
  - 若远程只有 IPv6，本地 socket 改用 `IPAddress.IPv6Loopback`
  - DNS 解析失败（`SocketException` 或空数组）统一转为 `InvalidOperationException`，消息包含无法解析的主机名
- `ConfigCommands.BuildAddCommand` — `--protocol` 选项增加默认值 `"tcp"`，未指定时不再抛异常

### 新增测试

- `ProxyServiceTests.StartUdpProxy_ForwardsData_ClientReceivesResponse` — UDP 数据报正向代理 + 回传响应
- `ProxyServiceTests.StartUdpProxy_UnresolvableHost_ThrowsInvalidOperationException` — 无法解析的主机名抛出明确异常
- `ConfigHandlersTests.HandleAdd_DefaultProtocol_DefaultsToTcp` — 未指定协议时默认为 Tcp
